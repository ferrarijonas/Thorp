CHECKLIST DE VALIDAÇÃO (ESTATÍSTICA INFERENCIAL)
=================================================

Após formular uma hipótese e testar no Python + MT5 + cruzar,
aplicar estes 7 passos ANTES de chamar de "edge":

┌─────────────────────────────────────────────────────────────┐
│  1. SIGNIFICÂNCIA ESTATÍSTICA                               │
│     O resultado pode ser sorte?                             │
│     - Teste t (p < 0.05)                                    │
│     - Teste de permutação (p < 0.05)                        │
│     - Se p > 0.05 → não é edge, é ruído                     │
├─────────────────────────────────────────────────────────────┤
│  2. COMPARAR VS INCONDICIONAL                               │
│     O edge é melhor que não fazer nada?                     │
│     - Retorno condicional > retorno médio da janela?        │
│     - Se não for melhor que aleatório → não é edge          │
├─────────────────────────────────────────────────────────────┤
│  3. CONSISTÊNCIA ENTRE METADES                              │
│     O resultado se sustenta em pedaços?                     │
│     - Dividir amostra em 2 metades (início/fim)             │
│     - Ambas positivas? Mesma direção?                       │
│     - Se uma metade for negativa → overfitting ou ruído     │
├─────────────────────────────────────────────────────────────┤
│  4. TESTE DE LIMIAR                                         │
│     O edge é sensível ao parâmetro exato?                   │
│     - Testar threshold +10% e -10% (ex: 0,27% e 0,33%)     │
│     - Se mudar muito o resultado → overfitting             │
│     - Se mantiver direção → edge mais robusto               │
├─────────────────────────────────────────────────────────────┤
│  5. ROBUSTEZ DE STOP/TARGET                                 │
│     O edge depende do SL/TP exato?                         │
│     - Testar SL ±20 pts                                     │
│     - Testar R:R 1:1, 1:2, 1:3                             │
│     - Se só funciona com SL exato → overfitting            │
├─────────────────────────────────────────────────────────────┤
│  6. PYTHON vs MT5                                           │
│     O backtest Python bate com o MT5?                      │
│     - Tolerância: até 15% de diferença no número de trades │
│     - Se diferença > 15% → calibrar .set ou investigar     │
│     - Se MT5 der negativo e Python positivo → falsa esperança│
├─────────────────────────────────────────────────────────────┤
│  7. DECISÃO                                                 │
│     Passou todos os filtros?                                │
│     - SIM (6/7 ou 7/7) → candidata forte → próximo: .set   │
│     - PARCIAL (4-5/7) → ajustar e re-testar                 │
│     - NAO (< 4/7) → morta. Próxima hipótese.               │
└─────────────────────────────────────────────────────────────┘

REGRAS DE OURO:
- Se p > 0.05 no teste t → NÃO É EDGE (mesmo que pareça bonito)
- Se as metades divergem → NÃO É EDGE
- Se o Python e o MT5 discordam → o MT5 manda
- Se o PF < 1.3 → não vale o risco
- Sharpe negativo com lucro positivo → provável viés de período único
